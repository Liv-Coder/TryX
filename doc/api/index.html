Tryx

A minimalistic and expressive Dart library for handling errors without traditional try-catch blocks.

Tryx provides a clean, functional, and beginner-friendly alternative using a declarative API similar to Result, Either, or safeCall patterns from Kotlin, Swift, or Rust.

✨ Features

  • 🛡️ Type-safe error handling - No more forgotten catch blocks
  • 🔗 Method chaining - Functional programming patterns with map, flatMap, fold
  • ⚡ Async support - First-class support for Future and Stream operations
  • 🔄 Retry policies - Built-in retry mechanisms with exponential backoff
  • ⏱️ Timeout support - Configurable timeouts for operations
  • 🎯 Custom error types - Define your own error hierarchies
  • 📊 Stream integration - Reactive programming with safe stream operations
  • 🌐 Global configuration - System-wide logging and error handling
  • 🧪 Comprehensive testing - 132+ tests ensuring reliability
  • 📚 Extensive documentation - Complete API docs and migration guides

🚀 Quick Start

Add Tryx to your pubspec.yaml:

dependencies:
  tryx: ^1.0.0

Basic Usage

import 'package:tryx/tryx.dart';

// Synchronous operations
final result = safe(() => int.parse('42'));
result.when(
  success: (value) => print('Parsed: $value'),
  failure: (error) => print('Error: $error'),
);

// Asynchronous operations
final asyncResult = await safeAsync(() => fetchUserData());
asyncResult
  .map((user) => user.name)
  .onSuccess((name) => print('Welcome, $name!'))
  .onFailure((error) => logError(error));

Method Chaining

final result = await safeAsync(() => fetchUser())
  .then((result) => result
    .map((user) => user.email)
    .flatMap((email) => safe(() => validateEmail(email)))
    .map((email) => email.toLowerCase())
  );

Custom Error Types

sealed class ApiError extends Exception {
  const ApiError();
  factory ApiError.network() = NetworkError;
  factory ApiError.auth() = AuthError;
}

final result = await safeWith<User, ApiError>(
  () => apiClient.getUser(),
  errorMapper: (e) => e is SocketException
      ? ApiError.network()
      : ApiError.auth(),
);

📖 Core Concepts

Result<T, E>

The foundation of Tryx is the Result type that encapsulates success and failure states:

  • Success<T, E>: Contains a value of type T
  • Failure<T, E>: Contains an error of type E
  • SafeResult<T>: Type alias for Result<T, Exception>

Safe Functions

  • safe(): Wraps synchronous operations
  • safeAsync(): Wraps asynchronous operations
  • safeWith(): Wraps operations with custom error mapping

Method Chaining

Results support functional programming patterns:

  • map(): Transform success values
  • flatMap(): Chain operations that can fail
  • when(): Pattern matching
  • fold(): Reduce to a single value
  • recover(): Handle failures with fallback values

🔧 Advanced Features

Retry Policies

final safe = Safe.network()
  .withRetry(RetryPolicy.exponentialBackoff(
    maxAttempts: 3,
    baseDelay: Duration(seconds: 1),
  ));

final result = await safe.execute(() => apiCall());

Stream Integration

final stream = Stream.fromIterable([1, 2, 3, 4, 5]);

await for (final result in stream.safeMap((x) => x * 2)) {
  result.when(
    success: (value) => print('Processed: $value'),
    failure: (error) => print('Error: $error'),
  );
}

Global Configuration

TryxConfig.instance
  ..defaultRetryPolicy = RetryPolicy.exponentialBackoff(maxAttempts: 3)
  ..globalLogger = (error) => print('Global error: $error')
  ..enablePerformanceMonitoring = true;

🎯 Use Cases

API Clients

Future<Result<User, ApiError>> fetchUser(String id) {
  return safeWith<User, ApiError>(
    () => httpClient.get('/users/$id').then(User.fromJson),
    errorMapper: (error) => ApiError.fromException(error),
  );
}

File Operations

SafeResult<String> readConfig(String path) {
  return safe(() => File(path).readAsStringSync())
    .recover((error) => '{}'); // Default config
}

Form Validation

Result<User, ValidationError> validateUser(UserInput input) {
  return safe(() => input.email)
    .flatMap((email) => validateEmail(email))
    .flatMap((email) => validatePassword(input.password))
    .map((password) => User(email: input.email, password: password));
}
