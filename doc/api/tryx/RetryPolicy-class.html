RetryPolicy

RetryPolicy class

Configuration for retry behavior in safe operations.

RetryPolicy defines how many times an operation should be retried and with what delay between attempts. It supports various backoff strategies including fixed delay, exponential backoff, and custom delay calculations.

Example:

// Simple retry with fixed delay
final policy = RetryPolicy(
  maxAttempts: 3,
  delay: Duration(seconds: 1),
);

// Exponential backoff
final exponentialPolicy = RetryPolicy.exponentialBackoff(
  maxAttempts: 5,
  initialDelay: Duration(milliseconds: 100),
  backoffMultiplier: 2.0,
  maxDelay: Duration(seconds: 10),
);
